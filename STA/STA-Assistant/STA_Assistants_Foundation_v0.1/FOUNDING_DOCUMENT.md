# STA Assistants — Documento Fundador

**Versión:** 0.1  
**Estado:** borrador para validación  
**Autor:** Mikel Zabal

## 1. Origen

ShapingTheAxe nació como un framework para estructurar el trabajo entre una persona y una inteligencia artificial. STA Assistants surge como una especialización de ese enfoque para la creación de asistentes de IA personalizados.

## 2. Problema observado

Crear un asistente serio para una empresa sigue siendo un proceso principalmente artesanal. Antes de construirlo hay que descubrir cómo funciona el negocio, qué problema merece automatizarse, qué reglas y excepciones existen, qué puede hacer el asistente, qué requiere aprobación humana y cómo se verificará su comportamiento.

## 3. Propuesta

STA Assistants pretende convertir conocimiento disperso de un negocio en una definición completa, diferenciada, trazable, verificable, aprobada y preparada para construcción.

## 4. Producto provisional

Sistema de producción asistida utilizado por un operador para descubrir el negocio, registrar hechos y contradicciones, modelar reglas, definir comportamiento, generar escenarios de aceptación y declarar si existe preparación suficiente para construir.

## 5. Usuario inicial

El usuario directo del primer MVP es el operador que define asistentes. Inicialmente puede ser Zabal. El responsable del negocio participa como fuente de información y autoridad de validación.

## 6. Resultado esperado

Una definición del asistente que permita a otra persona o sistema construirlo sin reconstruir toda la investigación original.

## 7. Diferenciación buscada

STA Assistants solo será valioso si demuestra resultados más rápidos, con menos omisiones, menos retrabajo, mayor personalización, mejor verificación y mejor handoff.

## 8. Relación con la empresa común

La empresa común de Alberto, Pedro y Zabal vende y opera asistentes para pymes. STA Assistants es una tecnología personal de Zabal que puede utilizarse como herramienta interna de producción.

## 9. Pregunta empresarial

Determinar si encaja mejor como herramienta interna, consultoría, servicio gestionado, producto para agencias, plataforma, licencia, white-label o SaaS.

## 10. Criterio de honestidad

Distinguir siempre entre visión, hipótesis, decisión, evidencia y resultado demostrado.
