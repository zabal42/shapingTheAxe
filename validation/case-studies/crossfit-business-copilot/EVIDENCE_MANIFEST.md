# Evidence Manifest

**Case:** `crossfit-business-copilot`
**Status:** Frozen

## Location

Evidence lives inside this case study, not in an external archive:

```text
validation/case-studies/crossfit-business-copilot/
├── docs/                 shared project input (brief, user stories, API discovery, acceptance criteria)
├── protocol/              experiment protocol, evaluator prompt, evaluation rubric
├── candidates/             candidate-a and candidate-b source trees, each with PROVENANCE.md
└── evidence/
    ├── evaluations/        three independent blind evaluation reports
    ├── evaluator-evidence/ scripts, fixtures, and raw outputs generated by evaluators
    ├── meta/                META_EVALUATION.md comparing the three reports
    └── private/             IDENTITY_MAP.md (candidate-x/y → candidate-a/b mapping)
```

The earlier consolidation used a single archive
(`crossfit-business-copilot-final-evidence.tar.gz`). That archive, and the
duplicate working copies that produced it (`packages/`, `blind-evaluation/`,
and a redundant `crossfit-evaluation-3/` run), have been superseded by this
directory structure. `candidate-x.tar.gz` was verified byte-identical
(SHA-256) across all three prior locations before removal; nothing was lost.

## Candidate source provenance

`candidates/candidate-a/` and `candidates/candidate-b/` are working trees
extracted from frozen git repositories, stripped of `.git`, `.venv`, and
caches. Their commit history and tags (`candidate-a-final`,
`candidate-b-final`, `crossfit-lab-v0.1`) are preserved as text in each
candidate's own `PROVENANCE.md`.
