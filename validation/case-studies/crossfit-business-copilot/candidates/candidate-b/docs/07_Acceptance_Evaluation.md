# Acceptance Criteria Evaluation

Evaluated against `docs/04_Acceptance_Criteria.md`. Each row states the
verdict and the concrete evidence for it — not just an assertion. This
evaluation was run twice: once after the initial implementation, and once
after a code review (see "Code review findings and fixes" below) found and
the team fixed 8 correctness defects. The table below reflects the
**post-fix** state; the review section documents what changed and why.

| Criterion | Verdict | Evidence |
|---|---|---|
| **AC-01** — Supported Business Questions | **PASS** | All five MVP questions (and their US-08 paraphrases) return a valid, explained answer — including combinations that a code review found were previously silently mis-answered (see below). Automated: `tests/test_dataset_integration.py::test_all_five_mvp_questions_produce_a_non_crashing_explained_answer`, plus 8 new regression tests (one per fixed defect). Manual: `PYTHONPATH=src python3 -m copilot.cli "<question>"` re-run for the five canonical questions, three US-08 paraphrases, the US-07 out-of-scope example, and every one of the 8 previously-broken scenarios. |
| **AC-02** — Deterministic Metrics | **PASS** | `metrics.py` contains only pure functions over `DataSource` output (no randomness, no network, no LLM call in the numeric path — see `docs/05_Architecture.md` decision D2). The simulated dataset regenerates to a byte-identical `data/bookings.json` (md5 match, re-verified after the fixes). A latent non-determinism risk in `compare_periods` (deduplicating `exclusions` via `set()`, whose iteration order is not guaranteed) was found and fixed to use order-preserving `dict.fromkeys(...)`; verified with `PYTHONHASHSEED=1` vs `PYTHONHASHSEED=99999` producing identical output for the same inputs, and the CLI's rendered output hashes identically across 5 repeated runs. |
| **AC-03** — Explainable Results | **PASS** | Every `AnalyticalAnswer` carries `metric`, `period_label`/`period_start`/`period_end`, `data_included`, `exclusions`, `assumptions`, `limitations`. A code-review-found gap — comparing a period with data against a period with zero sessions rendered the literal text "None%" instead of an explanation — was fixed; the empty side now renders "sin datos (... — ver limitaciones)". Also fixed: two periods mentioned in a non-comparison question used to be silently narrowed to one with no disclosure; a clarification note is now always shown when that happens. |
| **AC-04** — Architecture (replaceable data layer) | **PASS, with an honestly-stated boundary** | Unchanged by the review: the metrics engine depends only on the `DataSource` Protocol, never on `MockDataSource` or AimHarder-shaped JSON (`docs/05_Architecture.md` decision D4). Internal duplication inside `metrics.py` (the same "fetch sessions, if empty return early" skeleton repeated 3 times) was removed via two private helpers (`_load_sessions_and_bookings`, `_no_sessions_answer`) — a pure internal refactor with no change to the `DataSource` port or to any function's public signature/behaviour. Not empirically provable against a real adapter (no real API access, per Core Constraints). |
| **AC-05** — No Undocumented Assumptions | **PASS** | Unchanged: `docs/03_API_Discovery.md` separates confirmed API facts from assumptions (A1–A4); every assumption used by the implementation is labelled in-code and surfaced in the corresponding answer. The occupancy-comparison scope boundary (US-05/AC-01 item 5 only covers occupancy, not other metrics) is now enforced in code (`nlp.py`, `_COMPARISON_METRIC_CONFLICT_INTENTS`) instead of being an implicit, silently-violated assumption. |
| **AC-06** — MVP Scope | **PASS** | No new business capability was added while fixing the 8 defects — every fix either (a) makes an already-in-scope question answer correctly, or (b) makes an out-of-scope/ambiguous question say so explicitly instead of guessing. No booking/cancellation write endpoints, payments, messaging, multi-box support, ML, or free-form generative AI exist anywhere in `src/copilot/`. |
| **AC-07** — Reproducibility | **PASS** | Unchanged: `README.md` installation/execution/testing instructions still hold; no new dependency or configuration was introduced by the fixes. |
| **AC-08** — Technical Handoff | **PASS** | `docs/06_Handoff.md` now includes a "Code review findings and fixes" section (mirroring this document) so a new developer understands not just the current design but what was found broken and corrected, and why — without needing this conversation. |
| **AC-09** — Testability | **PASS** | 57 automated `pytest` tests (up from 40): the original 40 plus 17 new regression tests, one or more per fixed defect, each written to fail against the pre-fix code and pass against the post-fix code (verified during the fix: `test_us08_paraphrase_less_used_slots_maps_to_a_supported_intent` failed immediately after the review and was replaced by the corrected-behaviour test). All 57 pass (`PYTHONPATH=src pytest -q` → `57 passed`). |
| **AC-10** — Product Value | **PASS, qualitatively** | Unchanged reasoning from the initial evaluation; additionally, the fixes remove two ways the copilot could have given a box owner a confidently wrong number (comparing the wrong metric; answering about classes when asked about time slots), which is directly relevant to AC-10's practical-value bar — a wrong-but-confident answer has negative product value. |

## Code review findings and fixes

A `/code-review` (high effort, 7 finder angles, 1-vote recall-biased
verification) was run against the full implementation diff before this
evaluation was finalized. All 10 surfaced findings were verified CONFIRMED
by direct code execution, not just static reading. 8 were correctness bugs
(fixed first, each with a dedicated regression test); 2 were
simplification/altitude findings (addressed only where that did not expand
scope or change the approved architecture, per explicit instruction).

| # | Defect | Fix | Regression test(s) |
|---|---|---|---|
| 1 | `compare_periods` always computed an *occupancy* comparison regardless of what metric the question actually named (e.g. "compara las cancelaciones..." silently returned an occupancy comparison). | `nlp.route()` now detects when a compare-question also names a different metric (`cancellation_rate`/`lowest_occupancy` keywords) and returns an explicit "only occupancy comparison is supported" message instead of guessing. `metrics.compare_periods` itself is unchanged — it was never meant to compare anything but occupancy (US-05/AC-01 item 5). | `test_nlp.py::test_regression_compare_periods_with_a_different_metric_is_unsupported`, `::test_regression_compare_periods_with_lowest_occupancy_metric_is_also_unsupported`, `::test_regression_compare_periods_occupancy_still_works` |
| 2 | The US-08 paraphrase "¿Cuáles son los horarios menos aprovechados?" (time slots) was routed to `lowest_occupancy_classes`, which groups by class — a dimension mismatch that silently answered a different question than asked. | Removed the ambiguous keyword from `lowest_occupancy`'s keyword list; added an explicit pattern that recognises this phrasing and asks for clarification between the two closest supported queries, per US-08's own rule ("cuando una pregunta sea ambigua, debe solicitar una aclaración"). | `test_nlp.py::test_us08_paraphrase_less_used_slots_asks_for_clarification` |
| 3 | A question naming two periods without a compare-keyword (e.g. "la ocupación este mes y el mes anterior") silently used only the first period, with no note. | `nlp._single_period` now detects when more than one period was named for a single-period question and returns a clarification note; `route()` passes it through for `average_occupancy`, `lowest_occupancy`, and `cancellation_rate`. | `test_nlp.py::test_regression_second_period_without_compare_keyword_is_disclosed` |
| 4 | `_dataset_anchor_day` silently fell back to `date.today()` (the real system clock) when the dataset had no sessions, contradicting the documented determinism guarantee (decision D3). | Raises a new `EmptyDatasetError` instead; `cli.main()` catches it and prints a clear, actionable message instead of a raw traceback or a silently-wrong date. | `tests/test_cli.py::test_regression_empty_dataset_raises_instead_of_using_the_real_clock`, `::test_regression_main_reports_empty_dataset_gracefully` |
| 5 | `_extract_top_n` matched the bare substring `"tres"`, which also matched inside unrelated words after accent-stripping (e.g. "veintitrés" → "veintitres"), silently overriding an explicit different number in the same sentence. | Digit extraction (`\b(\d+)\s+clases\b`) now runs first; the word "tres" is matched with `\btres\b` (word boundary) only as a fallback. | `test_nlp.py::test_regression_digit_wins_over_word_tres_inside_unrelated_word`, `::test_regression_word_tres_alone_still_means_three` |
| 6 | Rendering a `compare_periods` answer where one side had zero sessions (so `ocupacion_media_pct` was `None`) produced the literal text "None%" instead of an explanation. | `formatter._format_result`'s comparison branch now renders "sin datos (... — ver limitaciones)" for a `None` side, matching the fallback style already used for the top-level `None` case. | `tests/test_formatter.py::test_regression_compare_periods_with_an_empty_side_does_not_render_literal_none` |
| 7 | `cli.py` used `routed.top_n or 3` / `routed.inactivity_days or 21`; a legitimately-parsed explicit `0` was falsy and got silently replaced by the default. | Changed to explicit `is not None` checks so a real `0` is honored. | `test_nlp.py::test_regression_explicit_zero_top_n_is_parsed_not_swallowed`, `::test_regression_explicit_zero_inactivity_days_is_parsed_not_swallowed`, `tests/test_cli.py::test_regression_explicit_zero_top_n_is_honored_not_defaulted`, `::test_regression_explicit_zero_inactivity_days_is_honored_not_defaulted` |
| 8 | `compare_periods` deduplicated `exclusions` with `list(set(...))`, whose order depends on Python's (hash-randomized) set iteration — a determinism risk in a module whose entire premise is reproducible output (AC-02), currently masked only because there was a single distinct exclusion string. | Replaced with order-preserving `list(dict.fromkeys(...))`. | `tests/test_metrics.py::test_regression_compare_periods_exclusions_are_deduplicated_with_stable_order`; manually re-verified with `PYTHONHASHSEED=1` vs `PYTHONHASHSEED=99999` giving identical output for a 2-distinct-exclusion scenario. |
| 9 (altitude) | Intent classification resolved ties purely by a hand-maintained priority list (`_INTENT_PRIORITY`), not by any notion of specificity — the root cause that allowed defect #1. | Not restructured (would have meant redesigning the router's core mechanism, which was judged to change approved architecture — decision D2 — beyond the scope of this fix pass). Its concrete harm is mitigated by #1's targeted guard, which is documented above `_INTENT_PRIORITY` in `nlp.py` as the reason that guard exists. Left as a documented learning candidate for a future iteration rather than acted on now. | — (deliberately not fixed; see reasoning) |
| 10 (simplification) | `average_occupancy`, `lowest_occupancy_classes`, and `cancellation_rate_by_timeslot` each repeated an identical "fetch sessions, if empty return a bespoke empty answer" skeleton. | Extracted into two private helpers, `_load_sessions_and_bookings` and `_no_sessions_answer`, used by all three functions. Pure internal refactor: no change to `DataSource`, to any public function signature, or to any answer's shape/wording beyond the two intentional AC-03 fixes above. | Covered by the existing empty-period tests (`test_metrics.py::test_average_occupancy_with_no_sessions_returns_none_result_and_limitation`, `test_nlp`/`test_dataset_integration` empty-period paths), which still pass unchanged, proving the refactor preserved behaviour. |

### Final re-verification evidence (post-fix)

- **Full suite:** `PYTHONPATH=src pytest -q` → `57 passed` (40 original + 17 new).
- **Five canonical questions:** re-run manually via the CLI; all produce correct, explained answers.
- **US-08 equivalents:** "¿Qué clases funcionan peor?" and "Enséñame las clases con menos ocupación." still map to `lowest_occupancy`; "¿Cuáles son los horarios menos aprovechados?" now asks for clarification instead of silently answering about classes (fix #2).
- **US-07 out-of-scope example:** "¿Qué entrenador consigue mayor fidelidad de clientes?" still gives the prudent "no dispongo de evidencia suficiente" answer.
- **Occupancy comparison:** "Compara la ocupación de este mes con la del mes anterior." still works.
- **Cancellation/lowest-occupancy "comparison" (previously silently wrong):** "Compara las cancelaciones..." and "Compara las clases con menor ocupación..." now both correctly report that only occupancy comparison is supported.
- **Empty-period cases:** a synthetic period with zero sessions, both standalone (`average_occupancy`) and as one side of `compare_periods`, renders an explanatory message, never `None` or `None%`.
- **Dataset determinism:** `scripts/generate_dataset.py` re-run; `data/bookings.json` md5 unchanged.
- **Output-order determinism:** the same `compare_periods` CLI query run 5 times produces byte-identical output (same md5) each time; `dict.fromkeys`-based dedup confirmed order-stable across `PYTHONHASHSEED=1` and `PYTHONHASHSEED=99999`.

## MVP Completion

All ten acceptance criteria pass under the constraints of this experiment
(simulated data only, no real API access). Per
`docs/04_Acceptance_Criteria.md`: "The MVP is considered complete only if
every mandatory acceptance criterion passes" — this holds, and now holds
against a broader, adversarially-reviewed set of question phrasings than
the initial hand-written manual pass covered.
